# Zentext 0.1.0-dev.3 release notes

Status: published prerelease.

- npm tag: `next`
- npm version: `0.1.0-dev.3`
- GitHub release: `v0.1.0-dev.3`
- Published: 2026-07-24

Zentext `0.1.0-dev.3` makes the local-first project-memory foundation directly
consumable by tools without terminal-text parsing. SQLite structured records
remain canonical. CLI, SDK, RPC, MCP, exports, and project-local skills are
views or adapters over the same validated state.

## Included

- formal schemas for externally supplied task, handoff, update,
  acknowledgement, export, and RPC inputs;
- control-character, Unicode, terminal-output, path, and payload sanitization;
- likely-secret detection before persistence, explicit local override
  recording, and output redaction across interfaces;
- actionable handoff-quality warnings;
- provenance for externally meaningful records;
- the storage-independent `MemoryStore` contract with SQLite as the canonical
  adapter;
- the typed TypeScript SDK;
- NDJSON RPC protocol `1.0` with typed methods and errors;
- read-only MCP access to the validated continuation;
- the project-local Zentext memory skill used by Codex and other local tools;
- compare-and-swap task revisions, atomic progress writes, and stale-handoff
  rejection; and
- deterministic GitHub CI on supported Node.js 22 and Node.js 24;
- deterministic project-memory search (`zentext search`, SDK `searchMemory`, RPC `memory.search`, MCP `memory.search`) across tasks, handoffs, decisions, blockers, validations, notes, referenced files, and provenance;
- explainable relevance and freshness ranking (issue #37 / PR #80);
- revision-aware process-local derived-page caching with a 64-entry, 4 MiB LRU, defensive copies, and cross-store invalidation (issue #38 / PR #81); and
- bounded search schema version `3` and MemoryStore interface version `1.3`.

The TypeScript SDK exports `openProject`, `getContinuation`,
`getActiveTask`, `validateHandoff`, `recordProgress`, `updateTask`,
`queryMemory`, and `searchMemory`. RPC protocol `1.0` exposes
`capabilities.get`, `project.open`, `continuation.get`, `task.active`,
`handoff.validate`, `progress.record`, `task.update`, `memory.query`, and
`memory.search`.

## Cross-tool evidence

One project installed from the packed package advanced from revision `2` to
revision `6` through four isolated environments:

- Codex Desktop;
- OpenClaw;
- Antigravity with Gemini; and
- Ollama through a controlled host.

Every participant recovered only recorded project memory, explained the state
before editing, completed one exact next action, advanced one revision, and
made the handoff it consumed stale. At revision `6`, CLI JSON, the TypeScript
SDK, NDJSON RPC, and MCP returned semantically equal canonical continuation
state. This is local field evidence, not a claim of native hosted-provider
integration.

## Runtime and package

- Supported Node.js range: `>=22.13 <25`.
- Release validation: Node.js 22 and Node.js 24.
- SQLite: normal `better-sqlite3` installation and install-script-disabled
  `node:sqlite` fallback.
- Public binaries: `zentext` and `zentext-mcp`.
- License: MIT.
- Publication tag when authorized: `next`.

The package allowlist includes runtime output, public documentation, the
portable demo, and release notes. It excludes source, tests, field-test
evidence and runners, local stores, credentials, environment files, and
tarballs.

## Known advisory

`npm audit` reports two moderate entries for `GHSA-frvp-7c67-39w9` through
`@hono/node-server` and `@modelcontextprotocol/sdk`. Zentext's shipped MCP
binary uses local stdio and does not import or start Hono's affected HTTP
static-file server. The advisory remains visible; no breaking dependency
change was made solely to silence it.

## Honest limitations

This Developer Preview does not provide:

- cloud synchronization, accounts, authentication, billing, or hosted storage;
- provider conversation migration or hidden model-state transfer;
- native hosted-provider plugins;
- graph, vector, semantic, or hybrid retrieval;
- multi-agent orchestration or concurrent-agent coordination;
- a permanent GUI; or
- MCP write tools.

Project-local memory remains fully usable and exportable without an account or
cloud service. Environment-specific presentation adapters (issue #35 / PR #66),
deterministic search (issue #36 / PR #79), relevance and freshness rules
(issue #37 / PR #80), and revision-aware caching (issue #38 / PR #81) are now
part of this prerelease. The M4 Deterministic Retrieval milestone is complete
through issue #38.
